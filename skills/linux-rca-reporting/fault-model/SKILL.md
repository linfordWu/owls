# 故障模式关联映射规则 (Fault Mode Association Rules)

本 Skill 旨在通过专家知识库，在识别出基础故障模式后，自动关联补齐其底层的依赖模式或共存模式。

## 1. 功能定义 (Functional Definition)
*   **目标**：输入一个或多个已识别的“起始模式 (Initial Mode)”，将其扩展为完整的“推荐关联模式集”。
*   **适用场景**：当已识别的故障模式涉及具有明确堆栈依赖关系（如存储堆栈）的组件时，提供确定性的补齐逻辑。

## 2. 映射规则 (Mapping Rules)

根据输入的“起始模式”，严格执行以下映射规则。

| 起始模式 (Initial Mode) | 关联补齐输出 (Associated Mode List) | 映射逻辑说明 |
| :--- | :--- | :--- |
| **文件系统故障** | `["文件系统故障", "磁盘故障"]` | 存储堆栈强相关性补齐 |
| **磁盘故障** | `["文件系统故障", "磁盘故障"]` | 存储堆栈强相关性补齐 |
| **其他模式** | `"无关联模型"` | **兜底逻辑 (Fallback)** |

### 强制执行要求 (Mandatory Requirements)：
1. **精准匹配**：一旦起始模式命中规则表中的特定项，必须输出表中定义的完整关联模式列表。
2. **唯一兜底**：若起始模式不在规则表中，**必须且仅能**输出字符串：`"无关联模型"`。
3. **原子性**：本 Skill 仅负责关联关系的静态映射，不负责后续的诊断推理或动作触发。

> [!IMPORTANT]
> **存储堆栈一致性规则**：针对“文件系统故障”与“磁盘故障”，系统必须视为原子整体。命中其中之一时，输出必须包含两者，以确保诊断覆盖层级的完整性。

---

## 3. 结果应用规范 (Usage Protocol)
1. **输出一致性**：输出结果应作为诊断链条的“专家确定性证据”。
2. **解耦原则**：调用方应根据返回的列表决定后续逻辑（如是否终止非相关的推断），Skill 本身不干预调用方的执行流程。
