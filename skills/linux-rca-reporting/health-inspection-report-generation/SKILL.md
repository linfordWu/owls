# 磁盘健康巡检预测分析和报告生成 Skill

本技能通过对前置节点收集的系统巡检数据（如 SMART 数据、RAID 状态、系统负载、文件系统状态等）进行分析，评估当前服务器磁盘的整体健康度，预测潜在的物理介质损坏和逻辑异常风险，并生成标准化的巡检与预测报告。

> **重要原则**：本 skill 与故障分析（RCA）不同，其核心在于“防患于未然”。分析需侧重于趋势预测、坏道劣化速度预警以及配置规范检查。

## 分析流程

### 阶段一：数据收集与状态标记 (Data Collection & Status Marking)

#### 步骤 1：输入数据解析

读取当前任务输入的1个或多个巡检结果数据文件。
提取收集的各类指标数据（包括但不限于：OS 层面的文件系统状态、磁盘 IO 延迟；底层 RAID 控制器状态；以及物理磁盘的 SMART 数据等），并归一化为内部检查项实体。

#### 步骤 2：时间基准与状态评估

- 提取各检查项的时间戳，统一时区，作为当前系统状态的“快照时间”。
- **强制约束**：时间必须格式化为绝对时间，如 `YYYY-MM-DD HH:MM:SS`。
- 将所有检查项基于历史基线和规范，分为以下状态类别：
  - ✅ **正常 (Healthy)**：SMART 指标在安全水位内，RAID 状态正常。
  - ⚠️ **警告 (Warning)**：指标逼近阈值（如 G-list 坏道开始增长，或 NVMe 温度偏高）。
  - ❌ **异常 (Critical)**：指标已超阈值（如触发不可修复错误），或文件系统变为只读。

---

### 阶段二：多文件综合趋势预测与风险分析

在拥有多个维度的巡检数据文件时，必须进行**跨文件综合分析**，并严格遵守以下分析方法论：

#### 1. 全面性与系统性 (Comprehensive Analysis)
- **拒绝信息孤岛**：必须遍历所有输入的巡检文件，绝不能遗漏任何一个磁盘、RAID 控制器或文件系统的检查结果。
- **全栈覆盖**：将 OS 层、控制链路层与底层物理 SMART 层指标综合在一个视图中考虑。

#### 2. 优先级与风险分级 (Priority & Stratification)
- **主次分明**：根据对数据安全的威胁程度，严格区分风险级别：
  - **致命故障/高危风险 (P0/P1)**：不可修复错误增加、介质彻底损坏、RAID 降级。
  - **亚健康状态 (P2/P3)**：坏道 (G-list) 开始爬升、链路偶尔重置、硬盘温度偏高。
- 报告中必须将高优问题置顶，不可让次要的维护建议淹没致命的风险。

#### 3. 跨维度关联与因果联系 (Cross-Correlation & Causality)
- **时间线关联**：若不同文件中的异常发生在相近的时间点（如 A 文件中发现 `link reset`，B 文件中出现 `fsck`），必须在分析中将其联系起来。
- **层级因果关联**：若底层出现介质坏道（L1），导致上层 IO 报错（L5），需要预测出这种“级联雪崩”的风险。
- **逻辑推断**：切忌只做简单的“阈值对比”，必须通过关联信息，推断出潜藏的根本性隐患（例如：温度升高 -> 物理介质加速老化 -> 坏道增加）。

#### 步骤 1：寿命与损坏速度预测
- 基于跨文件的时间序列数据或近期指标斜率，预测磁盘剩余寿命或彻底损坏的时间点。
- 例如：结合文件 A 的写入量与文件 B 的坏道增长曲线，预测出 `预计在 14 天后由于坏道超限引发离线`。

#### 步骤 2：隐患挖掘与风险链条
- 分析是否存在长期的隐性问题，如：内核日志中偶尔出现的 `I/O error` 记录、温度长期在临界值徘徊等。
- 将这些隐患与具体的业务挂载点关联，评估未来可能引发的“数据丢失传播链条”。

---

### 阶段三：输出标准健康巡检报告

巡检报告需客观、直观，并且对发现的隐患给出可操作的整改建议。
**最终报告必须以标准的 Markdown (.md) 格式输出。报告必须同时展示在对话界面，并写入本地的 Markdown 文件中。报告内的所有表格内容必须严格使用标准的 Markdown 表格格式进行展示，以保证在各类 Markdown 渲染器下的正确显示。**

## 二、分层检测体系

磁盘故障预测应覆盖 6 个层次，不能只看盘本体。
```text
┌────────────────────────────────────────────────────┐
│  L6  业务与存储服务层  OSD / EVS / ECS / 块存储       │
├────────────────────────────────────────────────────┤
│  L5  文件系统与OS层    dmesg / kernel / mount        │
├────────────────────────────────────────────────────┤
│  L4  控制器与链路层    RAID / HBA / SAS / CRC        │
├────────────────────────────────────────────────────┤
│  L3  槽位与环境层      Slot / 温度 / 背板 / 电源       │
├────────────────────────────────────────────────────┤
│  L2  寿命与负载层      上电时间 / 启停次数 / IO压力     │
├────────────────────────────────────────────────────┤
│  L1  盘本体 SMART 层   健康状态 / 错误计数 / 缺陷趋势   │
└────────────────────────────────────────────────────┘
            ↑  越往下越接近介质本身，越往上越接近业务影响
```
**预测重点**：L1 + L3 + 趋势维度，是提前发现故障最有价值的三个方向。
